Module Descriptions

  1. container Модуль містить клас Container, що відповідає за обробку контейнерів, їх типів, матеріалів та інших властивостей. Тут ви знайдете методи для отримання даних про контейнер, їх типи та матеріали.

  2. config Цей модуль містить конфігураційні налаштування бота, включаючи API токени, налаштування користувачів та питання для довідки. Це ядро для всього налаштування бота.

  3. callbacks У цьому модулі знаходяться функції для обробки зворотних викликів від користувачів. Використовується для інтерактивного спілкування з ботом, включаючи вибір варіантів меню та інших інтерактивних елементів.

  4. handlers Основні обробники, які реагують на повідомлення користувачів. Вони відповідають за обробку команд, запитів і переходів між різними секціями бота.

  5. user_data Модуль, який містить структуру для зберігання та обробки даних користувачів, таких як їх вибір, замовлення, персональна інформація.

  6. utils Додаткові утиліти для обробки даних, роботи з файлами, розрахунків та інших функцій, що допомагають боту працювати більш ефективно.

  7. main Основна точка входу для запуску бота. Цей модуль ініціалізує бота і починає процес його роботи.

  8. markups Модуль для створення різних клавіатур для взаємодії з користувачами. Клавіатури можуть бути як інлайн, так і звичайні, що дозволяють зручно управляти всіма функціями бота.

Getting Started

  1. Клонуйте репозиторій: git clone https://github.com/ваш-юзернейм/TGChatBot.git

  2. Встановіть всі залежності pip install -r requirements.txt

  3. Налаштування:

У файлі config.py у змінну TOKEN впишіть згенерований токен бота, а у змінну ADMIN_ID впишіть ID менеджера, якому будуть приходити повідомлення про нові замовлення.

Використання:

Запускаємо бота, через main.py, заходимо у бота, до якого Ви прив’язали свій токен, тепер Ви можеет користуватися основними функціями бота такими як: 1. Отримати відповіді на поширені запитання. 2. Отримати контакту інформацію. 3. Перейти на веб-сайт компанії, для перегляду каталогу товарів. 3. Сконфігурувати онлайн-замовлення, як забудовник ЖК, з прорахуванням кількості квартир та площі ЖК. 4. Замовити товар, як приватний покупець.
